# Use flat .txt files for local project memory

Status: Accepted
Date: 2026-04-09

## Context
The txtscape-mcp local tool needs a storage layer for project memory pages.
We considered SQLite and PostgreSQL, but binary SQLite files in git are a
blocker for diffing, merging, and code review. A database would add
external dependencies to what should be a zero-dep CLI tool.

## Decision
Store pages as plain .txt files in .txtscape/pages/. One file per page.
Folders map to path-like structures. No database — the filesystem IS the
storage layer.

## Consequences
- Pages are diffable and mergeable in git
- Zero external dependencies (no database driver)
- Multi-user collaboration works through normal git workflows
- No query language — search is grep-based
- 1MB max file size keeps things reasonable
