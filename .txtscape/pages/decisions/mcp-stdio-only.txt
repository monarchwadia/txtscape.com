# MCP stdio transport only for txtscape-mcp

Status: Accepted
Date: 2026-04-09

## Context
The txtscape-mcp local tool needs a transport for communicating with AI
agents/IDEs. HTTP transport would require a running server process, port
management, and background daemon lifecycle.

## Decision
txtscape-mcp uses stdio transport only. The agent/IDE launches the binary
as a subprocess and communicates via stdin/stdout JSON-RPC. No ports, no
daemon, no Docker.

## Consequences
- Zero infrastructure — just a Go binary
- VS Code launches it automatically via mcp.json
- No auth needed — filesystem permissions are sufficient
- Future: can add HTTP transport if needed for web UI or remote access
