# Integration Tests

## Scope

Test a single API endpoint end-to-end against a real database.
Request -> router -> handler -> database -> response.

## Build tag

  //go:build integration

Run with: go test -tags=integration ./...

## What to test

1. Auth endpoints: POST /signup, POST /login — status codes, token issuance, errors
2. Page CRUD: PUT/DELETE/GET — content storage, retrieval, deletion
3. Database constraints: CHECK constraints fire correctly
4. Application limits: file counts, folder depth, size limits
5. Auth middleware: token validation on protected endpoints
6. Directory listings: auto-generated when no index.txt

## Edge cases specific to integration

- Concurrent writes to the same path
- Cross-user namespace protection
- Token from one user used for another user's space
- DELETE a file that doesn't exist (404, not 500)
- Empty body on PUT
- Database constraint violations
