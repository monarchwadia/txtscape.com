# DB Test Setup

Database lifecycle for integration and E2E tests.
Note: txtscape-mcp does NOT use a database — filesystem only.
This reference applies to the original txtscape server.

## Strategy

Each test PACKAGE gets its own temporary Postgres database.
Tests within a package run sequentially (no t.Parallel()).

## TestMain Pattern

1. Connect to Postgres using DATABASE_URL
2. Create a unique database (test_packagename_timestamp)
3. Connect to the new database
4. Run migrations
5. Run tests
6. Cleanup (DROP DATABASE) — always, even on failure

## Shared Helper

testutil.SetupTestDB(t) creates fresh DB, runs migrations, returns pool + cleanup.

## Why per-package, not per-test

- Packages already run in parallel (go test ./... parallelizes across packages)
- Tests within a package are sequential by default — no conflicts
- Creating a DB per test is slow (~100ms per CREATE DATABASE)
- If isolation needed within package, use DELETE FROM tables between tests

## Running

  go test ./...                                           # unit only, no DB
  DATABASE_URL=... go test -tags=integration ./...        # integration
  DATABASE_URL=... go test -tags=e2e ./e2e/               # e2e
