# E2E Tests

## Scope

Multi-step user journeys. Each test tells a story with business meaning.

## How E2E differs

- Unit: Single function, mocked deps
- Integration: Single endpoint, real DB
- E2E: Multi-step journey, real DB/filesystem

## Build tag

  //go:build e2e

Run with: go test -tags=e2e ./e2e/

## Test documentation

Every E2E test has a narrative comment:

  // Journey: New user publishes their first page
  // Business context: This is the core use case.
  // Steps: 1. Sign up  2. PUT page  3. GET page  4. Verify listing
  // Expected: All steps succeed. Page is publicly readable immediately.

## Journeys to cover

Publishing: first page, folder organization, update existing, delete, custom index
Auth: cross-user protection, multiple tokens, wrong password
Limits: file too large, folder depth, files per folder
Browsing: cross-user links, subdirectory listing
