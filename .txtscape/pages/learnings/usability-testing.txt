# Usability Testing Learnings

## Key findings from first parallel agent usability test (April 2026)

### list_pages output
- Default call (no recursive) returns folder icons only: `📁 rooms/` — not files
- Recursive call returns flat emoji list: `📄 rooms/entrance-hall.txt  —  # Title`
- Tutorial must show real emoji format, not invented tree-view
- When pages dir doesn't exist (fresh project), server returned error — fixed to return "(empty directory)"

### Context engineering for nickname/txtscape alias
- Agent's initialize instructions used "journal" exclusively, disconnected from "txtscape" (which users write naturally)
- Fix: add explicit alias sentence — "When the user says 'journal' or 'txtscape'... they mean this tool"
- Test: check instructions contain `"journal" or "txtscape"` as a phrase, not just either word separately

### Git workflow
- Tutorial must integrate git commits at every step — not just mention it in "What's next"
- page_history only works if the user has been committing; tutorial should say so explicitly
- Starting git in step 1 enables page_history to work by step 6

### Template placeholder {name}
- Server does NOT substitute {name} — it's advisory text the LLM reads and interprets
- Tutorial must clarify this to prevent junior devs from expecting server-side substitution

### _meta.hash
- Returned by put_page, get_page, append_page, str_replace_page, move_page
- Agent manages this automatically for optimistic concurrency
- Brief tutorial callout prevents confusion when developer sees it in raw tool responses
