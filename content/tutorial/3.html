Summoning Code

One room in memory. Time to make the AI build a game from it.

Generate the game

You have a tavern described in txtscape. Now ask the agent to turn it into a playable game:

Tell your agent
"Read the rooms/tavern page from txtscape. Then build a text adventure game in index.html — a single HTML file with inline CSS and JavaScript. Use a dark theme with a monospace font. Show the room description, list any items the player can pick up, and show exits as clickable links. Start the player in the tavern."

Watch what the agent does:

  1. Calls get_page to read rooms/tavern.txt
  2. Parses the room description, exits, and items from the content
  3. Generates index.html with a working game

Open it

Open index.html in your browser. You should see a dark-themed text adventure showing:

  • The Rusty Flagon's description — candles, bartender, ale and smoke
  • The rusty sword — available to pick up
  • A "Dark Forest" exit — clickable but leads nowhere (there's no forest room yet)

The exact layout depends on your agent's choices. That's fine — the important thing is that the content matches what you wrote in your txtscape page.

The key insight

The AI didn't invent the tavern. It read your description from txtscape and used it faithfully. The room text, the items, the exits — all came from your page.

Your txtscape pages are the source of truth. The code is derived from them.

Without txtscape, you'd describe the tavern in chat, the AI would generate code, and next session it would forget everything. You'd re-describe the tavern, get slightly different details, and end up with inconsistencies.

With txtscape, the knowledge persists. Every session starts from the same source.

Try this: Start a new chat session (close and reopen the AI chat). Ask the agent "What rooms exist in this project?" — it can call list_pages and find the tavern without you re-explaining anything.
