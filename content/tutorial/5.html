The Living World

Change a page. The AI adapts everything to match.

Change how the cave works

The Crystal Cave is too easy — the player just walks in and takes the gem. Let's make it more interesting. Ask the agent to update the page:

Tell your agent
"Update the rooms/cave page in txtscape. The cave has partially collapsed. The glowing gem is now trapped behind rubble — the player needs the rusty sword to clear it. Add a new exit: down leads to the Underground River."

The agent uses str_replace_page or put_page to update the cave. The page now reflects the new reality.

Add the new room

Tell your agent
"Create a txtscape page at rooms/river.txt:

The Underground River flows through a carved stone channel. The air is cold and damp. Ancient runes glow faintly on the walls, casting blue light across the dark water. The river disappears into darkness downstream.

Exits: up leads to the Crystal Cave.

Items: a rune-etched key rests on a stone ledge above the water."

Your game world just grew:

Tavern --north--> Forest --east--> Cave --down--> River

Regenerate

Tell your agent
"Read all txtscape pages with snapshot. The cave has changed and there's a new room. Update index.html so the cave shows the collapse, the gem requires the rusty sword to retrieve, and players can explore the Underground River."

Open index.html. The game has changed:

  • The cave description mentions the collapse
  • The gem can't be picked up without the sword
  • A new "Underground River" exit appears in the cave
  • The river room works — runes, key, navigation back up

The aha moment

You edited a text file. The entire game adapted.

This is the power of persistent, structured memory:

  • You edit knowledge, not code. The game world lives in txtscape pages. The code is derived from them.
  • The AI stays consistent. It reads the current state of all pages every time you ask it to generate code. No stale context.
  • Changes propagate. Update one page, regenerate, everything aligns. The cave collapse, the sword requirement, the new exit — all reflected automatically.

No more "the AI forgot the cave has a gem" or "the forest description changed but the game still shows the old one." The pages are the source of truth. The code follows.

Try this: Open your .txtscape/pages/ directory in your editor. Edit a room's description directly — change a color, rename an item, add an exit. Then ask the agent to regenerate. It reads the updated pages and the code matches. Human edits and agent edits are interchangeable.
